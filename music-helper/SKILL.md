# Music Helper

## 快速参考

### 命令列表

```bash
# 搜索
node index.js --search "关键词"           # 搜索音乐
node index.js --search "关键词" --page 2  # 搜索指定页

# 购物车
node index.js --add 1,3,5              # 添加到购物车
node index.js --cart                     # 查看购物车
node index.js --remove 1,2             # 从购物车移除
node index.js --clear                   # 清空购物车
node index.js --checkout --dir "路径"   # 统一下载

# 直接下载
node index.js --download 1,3 --dir "路径"  # 直接下载（不经过购物车）
```

### 参数说明

| 参数 | 说明 |
|------|------|
| `--dir` | 下载目录（必填），默认 `{项目根目录}/downloads` |
| `--page` | 搜索页码（从1开始） |
| `--search` | 搜索关键词 |
| `--add` | 购物车序号（1-based，如 1,3,5） |
| `--download` | 缓存序号（1-based） |

### 限制说明

- 每页搜索结果：10条
- 购物车最大容量：50首
- 缓存文件：`../cache/search_cache.json`、`../cache/cart.json`

## 对话流程

### 智能流程（推荐）

AI 根据用户选择自动决定使用购物车还是直接下载。

**搜索后引导：**
展示搜索结果表格后，AI 应引导用户：

```
🎵 找到 10 首结果：

| # | Title | Artist | Album | Format | Size |
|---|-------|--------|-------|--------|------|
| 1 | 倔强 | 五月天 | 倔强 | FLAC | 28.7 MB |
|...

请问：
- 输入序号直接下载（如 1,3,5）
- 输入"加购物车"暂存，稍后一起下载
- 输入"更多"查看更多结果
```

**用户响应处理：**
- 用户输入序号 → 询问下载目录 → 执行下载
- 用户说"加购物车" → 添加到购物车 → 询问"还需要其他歌曲吗？"
- 用户说"更多" → 执行翻页

**添加购物车后：**
```
已加入 3 首歌曲到购物车。请问：
- 继续搜索添加更多歌曲
- 输入"查看购物车"确认清单
- 输入"下载"开始下载
```

**查看购物车：**
```
🛒 购物车（5首）：

| # | Title | Artist | Album | Format | Size |
|---|-------|--------|-------|--------|------|
| 1 | 倔强 | 五月天 | 倔强 | FLAC | 28.7 MB |
|...

确认下载吗？输入"下载"开始，或"移除X"调整。
```

## 异常处理

### 1. Cookie 失效
- **现象**：搜索或下载报错
- **处理**：脚本自动刷新 Cookie 并重试

### 2. 缓存为空或过期
- **现象**：`cache is empty` 错误
- **处理**：提示用户重新搜索

### 3. 下载失败
- **现象**：`Failed: X song(s)`
- **处理**：检查网络，重新下载失败曲目

### 4. 购物车已满
- **现象**：添加时提示 `Cart is full`
- **处理**：提示用户先下载或清空部分歌曲

### 5. 索引超出范围
- **现象**：序号 > 结果数量
- **处理**：提示有效序号范围

### 6. 网络错误
- **现象**：请求超时或连接失败
- **处理**：检查网络，重试
